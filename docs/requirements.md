# 要件定義書 家族向け予定・家計簿アプリ（PlannerwithExpense）

---

## ① 使用目的

家族の「予定」と「支出」を一つのアプリで一元管理したい。これまで別々に管理していた家族のスケジュールと支出を一箇所にまとめ、把握しやすくすることが目的。

---

## ② 対象ユーザー・ユースケース

**対象ユーザー：** 家族メンバー（それぞれ自分のアカウントを持つ）

**ユースケース：**

| ユースケース | 機能要件 |
|------------|---------|
| 家族の予定をカレンダーで確認・登録・編集・削除する | F01, F02, F03, F04 |
| 支出を記録・確認・編集・削除する | F05, F06, F07, F08 |
| 自分のアカウントでログイン・ログアウトする | F09, F10 |

## ⑤ 機能要件

### ユーザー認証機能

| ID | 機能名 | 内容 |
|----|--------|------|
| F09 | アカウント登録 | メールアドレス＋パスワードで家族メンバーがアカウントを作成できる |
| F10 | ログイン／ログアウト | 登録したアカウントでログイン・ログアウトできる |

### 予定管理機能

| ID | 機能名 | 内容 |
|----|--------|------|
| F01 | カレンダー表示 | 月・週単位で予定を一覧表示できる |
| F02 | 予定登録 | 新しい予定を登録できる |
| F03 | 予定編集 | 登録済みの予定を編集できる |
| F04 | 予定削除 | 登録済みの予定を削除できる |

### 家計簿機能

| ID | 機能名 | 内容 |
|----|--------|------|
| F05 | 支出登録 | 金額・日付・カテゴリ・メモを入力して支出を記録できる |
| F06 | 支出一覧表示 | 記録した支出を一覧で確認できる |
| F07 | 支出編集 | 登録済みの支出を編集できる |
| F08 | 支出削除 | 登録済みの支出を削除できる |

### 次フェーズ以降（バックログ、MVPには含まない）

- 予定と支出の紐付け
- 「誰の予定/支出か」の色分け・区別表示
- 家族間の支出共有・集計（月合計、メンバー別など）
- 予算設定と超過アラート

## ⑥ 技術スタック（開発環境は基本設計に記載）

前回プロジェクト（`task-manager-personal`: React + Java/Spring Boot + PostgreSQL）との差別化として、以下に変更する。

### 1. フロントエンド

| 項目 | 技術 |
| --- | --- |
| フレームワーク | Vue.js（SPA） |
| ビルドツール | Vite |

### 2. バックエンド

| 項目 | 技術 |
| --- | --- |
| 言語 | Python |
| フレームワーク | FastAPI |
| 認証 | 自前実装（JWT、メール＋パスワード） |
| API形式 | REST API |

### 3. データベース

| 項目 | 技術 |
| --- | --- |
| RDBMS | MySQL |

### 4. インフラ・開発ツール

| 項目 | 技術 |
| --- | --- |
| ローカル開発 | Docker Compose |
| デプロイ先 | AWS（無料枠の範囲。主に課題提出用の公開先という位置づけで、日常利用はローカル運用を想定） |
| バージョン管理 | Git + GitHub |
